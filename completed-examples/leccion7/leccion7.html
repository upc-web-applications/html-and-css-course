Recomendaciones y errores comunes

En esta lección aprendemos buenas prácticas para escribir HTML y CSS, además de errores comunes que suelen cometer los principiantes.

¿Por qué es importante escribir buen código?

Cuando estamos aprendiendo desarrollo web, no solo importa que la página funcione. También es importante que el código sea ordenado, fácil de leer y fácil de corregir.

Un código bien escrito ayuda a encontrar errores más rápido y permite que otras personas también puedan entenderlo.

Buenas prácticas al comenzar

1. Ordena tu código

Usa espacios, saltos de línea e indentación. Esto hace que el código sea más fácil de leer y entender.

2. Usa nombres claros

Cuando crees clases en HTML y CSS, usa nombres que expliquen su función. Por ejemplo, es mejor usar tarjeta que usar caja1.

3. Prueba poco a poco

No escribas todo el código de una sola vez. Primero crea una parte pequeña, revisa si funciona y luego continúa agregando más elementos.

4. Usa comentarios

Los comentarios sirven para explicar partes importantes del código. En HTML se escriben así: <!-- comentario -->.

Errores comunes para principiantes

1. Olvidar cerrar etiquetas

Algunas etiquetas necesitan apertura y cierre. Por ejemplo: <p>Hola</p>. Si olvidamos cerrar una etiqueta, la página puede verse desordenada.

2. Escribir mal una propiedad CSS

CSS necesita que las propiedades estén bien escritas. Por ejemplo, se escribe background-color, no backgroud-color.

3. Olvidar el punto en las clases

En HTML escribimos class="tarjeta", pero en CSS seleccionamos esa clase usando un punto: .tarjeta.

4. Copiar código sin entenderlo

Copiar código puede ayudar al inicio, pero es importante leerlo, modificarlo y entender qué hace cada parte.

Ejemplo rápido

Mala práctica

Usar nombres poco claros como caja1, texto2 o div3.

Buena práctica

Usar nombres más descriptivos como tarjeta, encabezado o boton-principal.

Checklist antes de terminar una página

  • ¿Mis etiquetas HTML están bien cerradas?
  • ¿Mi código está ordenado y fácil de leer?
  • ¿Mis clases tienen nombres claros?
  • ¿Probé los cambios poco a poco?
  • ¿Revisé si escribí correctamente las propiedades CSS?
